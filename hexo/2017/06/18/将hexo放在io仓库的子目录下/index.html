将hexo放在io仓库的子目录下

  然后，每次执行hexo d时，直接把public文件夹中生成的内容push到io仓库的对应分支上了。但是有小伙伴发现执行hexo d时，特别的简单粗暴，因为它直接把你之前io仓库的内容给覆盖了，那我之前的内容就没有了，那如何在保留我io仓库原本的内容的同时，将博客也push到GitHub上呢？这里我是把博客push到了io仓库的子目录了，感谢小伙伴赵py的帮助！

设置io仓库的本地仓库

  首先，将你GitHub上的io仓库clone到本地。

  在电脑的任意一个位置创建一个文件夹，用来存放GitHub上的仓库(我在E盘下创建了GitHub文件夹)。打开该文件夹，右击git bash here打开git命令行。

  接下来，获取你的GitHub上io仓库的链接，打开你的GitHub的io仓库，在右边页面可以看到下图界面，可以选择使用SSH或者HTTPS，然后相应的链接出现在下框中，单击链接右边的图标，此时链接已经复制到了剪贴板上了。

  最后在git命令行中执行如下命令，就可以将你的io仓库复制到本地了。
git clone git@github.com:sylviaxgj/sylviaxgj.github.io.git

  clone成功以后，在你创建的文件夹下（我的是GitHub），会有一个名为你的GitHub名.github.io的文件夹（以下简称为io仓库，我的是sylviaxgj.github.io），里面包含了你GitHub上io仓库中的内容。现在我们要在保留这些内容的前提下，将hexo博客发布到io仓库中。

  进入你的GitHub名.github.io的文件夹（我的是sylviaxgj.github.io），然后，新建一个文件夹（我的是hexo）用来存放你的hexo博客的内容。

生成静态文件

  在hexo的根目录下，打开git，执行hexo clean(清除缓存)，执行成功后，然后再执行hexo g（生成静态文件）。上述命令执行成功后，在你本地的io仓库下的hexo文件夹下，就会生成一堆文件，就是将来要上传到GitHub上的静态文件。

使用Git来上传

  到此时，本地仓库中既有了你之前的内容，而且还有了hexo博客的内容，而且还链接起来了，那怎么把这些内容上传到GitHub上呢？使用Git！

git切换分支

  在本地io仓库下，打开git命令行，我先将本地仓库的分支切换为develop分支，将提交的内容都push到develop分支上，这样保证master分支，随时都是正确的版本或者页面，在develop分支上测试无误以后，再合并的master分支上。切换以后，在git命令行当前路径的后面就有一个括号，里面表示当前分支的名字。


  若没有develop分支，则使用git checkout -b 分支名 来创建并切换到该分支。

git add

  git add是用来将文件添加到暂存区的，这里我每次都使用git add .（注意这后面与个点），表示将当前所有内容提交到暂存区。


git status

  git add .执行完以后，可以使用git status来查看当前git仓库的状态（当然这一步可以不用执行，只是为了检查，建议执行）：



  其中显示了你距离上个版本的改动，表示已经在暂存区，需要提交的内容。

git commit

  接下来，就是把上述已经在暂存的内容提交，执行git commit -m "此处填写你距上一次的改动"。



  执行完成以后，就将改动提交到了仓库的版本区（即.git文件）。

git push

  最后，执行git push，将本地仓库的内容push到GitHub上的远程仓库中

在GitHub上合并分支

查看新的push分支

  登录你的GitHub，然后进入io仓库，你会看见显示一条带浅色背景的文字，表示你最新push的分支，然后点击右侧的Compare&pull request，进行查看。

  若是没显示，则将分支切换到develop分支，然后点击旁边的New pull request,自己进行查看。


Open a pull request

  这一步表示，新建了一个合并分支请求，这里可以填写，你与上一次版本的改变的一个简单的描述。填写完以后，点击Create pull request。


合并分支

  这一步会对上一步的合并请求进行检查，若是检查无误则可以允许合并，这里一般是仓库的拥有者来允许合并的，在本文中我们对自己的仓库发起了New pull request，所以自己进可以合并了。若是对别人的仓库fork，然后clone到本地修改，最后发起New pull request时，则是仓库的拥有者才有权限进行允许合并。

  检查无误以后，点击Merge pull request即可。


确认合并

  最后一步，点击*Confirm merge即可。



  完成以后，进入（https://你的GitHub名字.github.io/ ）查看就行了。

注意

   切记！！以后禁止使用hexo 的命令！因为，使用hexo d命令会直接把博客内容push到io仓库的根目录（按照在站点配置文件中写的那样），从而覆盖了io仓库的其他内容

git常用命令总结

命令 解释
git add file 将文件file添加到暂存区
git commit -m “提交的简单描述” 将暂存区的内容提交到本地仓库
git status 查看当前仓库中的状态
git commit -am “提交的简单描述” git add和git commit -m “信息”的合并
git push 将本地仓库推到远程仓库的对应分支上
git branch 分支1 创建分支1
git checkout 分支1 切换到分支1
git checkout -b 分支1 创建并切换到分支1
git push origin –delete 分支1 删除远程分支1
git branch -d 分支1 删除本地已合并分支
git branch -D 分支1 强制删除本地未合并分支1
git clone 仓库的地址 clone分支到本地
