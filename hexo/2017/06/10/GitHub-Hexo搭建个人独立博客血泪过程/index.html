GitHub-Hexo搭建个人独立博客血泪过程

发表于 2017-06-10 | 阅读次数 | 本文总阅读量 次

这是我的第一篇博客文章，想想还是有些小激动的。下面来跟大家分享一下，我搭建博客的详细过程和一些小曲折。总体来说，还算是顺利吧！

准备软件工具

  • git客户端

    作用：把本地的hexo内容提交到GitHub上去,我的版本是git@2.12.2（git version命令）
  • node.js
    作用：用来创建hexo博客框架的，我的版本是node.js@v6.10.3（node -v命令）
  • GitHub账号

    使用github Pages来托管GitHub上的静态网页，用它来搭建博客

软件安装和配置

本文是在win10系统下安装的
下面介绍详细的安装过程：

 建议先安装git。因为用node.js安装hexo的时候，可以用git。

git安装

  安装git for windows。

  安装好了之后，直接在电脑中任意一个文件夹下，右键，就可以看到如下选项：



然后，选择Git Bash Here就可以在当前路径中打开git的命令行界面了。

GitHub设置

GitHub账号注册

  进去GitHub的官网进行注册，github传送门。

  注册过程比较简单，当然，必须记住你的邮箱和密码。注册过的童鞋请跳过，我之前就注册过了，这一步就跳过了。

创建GitHub仓库

  注册号账号以后，就是创建仓库了，用来放置你的代码文件。进入GitHub后，点击左上角的“+”号，如下：


进入到创建仓库页面，创建仓库时，只需要填写仓库名字，其他的不用管，然后点击“Create repository”即可，这里注意，仓库名一定要是yourname.github.io,比如我的GitHub昵称为sylviaxgj，则我的仓库名字就是sylviaxgj.github.io。（我之前已经创建过了，所以会有已存在警告，创建的时候是不会有的）

配置SSH Keys

检查SSH Keys

  这一步的目的是使本地git项目与远程的GitHub建立联系，使本地git提交时，拥有远程GitHub的权限，来提交到你的GitHub上。

  首先,来检查一下，你的电脑上的ssh key打：

在电脑任意文件夹处，右键git bash here，

然后在命令行输入cd ~/.ssh
来检查本机的ssh密钥

  如果提示:No such file or directory ，说明是第一次使用git。否则，会进入到.ssh文件夹，这个时候，你可以输入

pwd

来显示当前的路径

找到当前路径后，打开路径，会发现有三个文件，分别是id_rsa、id_rsa.pub、known_hosts。其中id_rsa是私钥，id_rsa.pub是公钥。可以直接删除.ssh文件夹中的所有文件，重新配置。也可以使用其中的文件。

生成新的SSH Key

  打开git，在命令行下，输入下列命令：
ssh-keygen -t rsa -C "你GitHub的注册邮箱l@example.com"，
注意：“C”为大写。

  然后，回车后，会让你输入密码，这个密码是在你每次提交项目时输入，这样可以防止别人往你的项目里提交内容。如何密码为空的话，提交项目时，不用输入密码。若是不需要密码，可以直接三次回车。

  此时，已经生成新的SSH Key了，接下来，就是向GitHub中添加SSH Key了。

向GitHub中添加SSH Key

  在本地设置了SSH Key，就可以添加到GitHub上了。

  打开GitHub登录你的账户，然后点击右上角的头像，选择Settings,在显示页面的左边选择“SSH and GPG keys”，然后右上角点击“New SSH key”来创建一个SSH key。

  填写SSH key时，只用填写“Title”和“Key”两项。“Title”可以随便起名字，但是建议最好选一个反映本机相关信息的名字，表示这个SSH key是和本机相连。“Key”中填入上一步生成的id_rsa.pub中的内容，id_rsa.pub的路径一般是在C:\Users\你的电脑名.ssh文件夹下（我的是在C:\Users\xgj.ssh）。最后，点击“Add SSH key”按钮即可。
;

测试

  测试设置是否成功，可以输入ssh -T git@github.com，
来验证是否连接成功，（注意，这里就输入该命令，不用更改成自己的账户），如果出现警告，直接输入“yes”就好。

  如果显示Hi xxx! You've successfully authenticated,but GitHub does not provide shell access.，则表示连接成功。

Hexo安装

Hexo介绍

   Hexo 是一个简单地、轻量地、基于Node的一个静态博客框架，可以方便的生成静态网页托管在github和Heroku上，引用Hexo作者 @tommy351 的话：

快速、简单且功能强大的 Node.js 博客框架。A fast, simple & powerful blog framework, powered by Node.js.

  由于github pages存放的都是静态文件，博客存放的不只是文章内容，还有文章列表、分类、标签、翻页等动态内容，假如每次写完一篇文章都要手动更新博文目录和相关链接信息，相信谁都会疯掉，所以hexo所做的就是将这些md文件都放在本地，每次写完文章后调用写好的命令来批量完成相关页面的生成，然后再将有改动的页面提交到github。

Hexo安装

  注意：安装Hexo过程中最好都用git来安装。
  先将之前安装的node.js安装好，然后在任意位置打开git，命令行中输入以下命令：

npm install -g hexo

  这个过程可能有点久，稍等一会。

Hexo初始化

  在电脑中创建一个文件夹，这个文件夹将会保存我们以后写的blog的内容。这里我建立了一个名为hexo博客的文件夹。然后在此文件夹中，打开git，执行下列命令：

hexo init


  当看到“INFO Start blogging with Hexo!”，则表示初始化成功。

  然后，Hexo就会在你这个文件夹中创建网站所需的所有文件。

端口占用情况解决

  当使用hexo s来开启本地预览的时候，并不是一直都这么顺利，有时候会出现端口占用情况，此时，命令行会显示出FATAL Port 4000 has been used.Try other port instead 。
  此时，可以打开Windows的cmd，输入netstat -aon|findstr "4000"，得到如下结果

  在上图中最后一列可以看到一个PID（这里是10928），表示当前占用4000端口的进程的PID。

  然后打开Windows的任务管理器，在详细信息选项页面，找到对应的PID，并将其停止。重新执行hexo s即可。

  至此，Hexo的安装就完成了，下面介绍Hexo的使用和一些配置。

使用Hexo

  上述过程，只是在本地生成了博客的页面，还没有把它发布到GitHub上呢。

根目录相关文件介绍

  在你的hexo目录（我的是“hexo博客”文件夹）下，存放着初始化网址生成的一系列文件。其目录结构如下：

_config.yml文件

   这个_config.yml文件在根目录下，我们称它为站点配置文件。在themes文件的各个主题中的_config.yml文件，称为主题配置文件。

   在站点配置文件中，可以修改网页的大部分参数。而在主题配置文件中则包含了相关主题的配置参数等。

scaffolds文件

   模板文件夹。其中包含post.md文件，在新建文章时，就是根据post.md文件中的模板来创建的。可以修改其中的参数，来创建你喜欢的模板。
   当然，若是需要多个模板，也可以自己创建模板文件，并保持为md文件。使用特定模板的命令为，hexo new 模板文件的名字 “文章名字”（后面讲这个命令）。

source文件

   当用户新建了网页或者文章时，在source文件下，就会生成相应的内容。新网页直接放在source文件夹下，文章放在source文件夹下的_posts文件夹下。相当于草稿，你可以在本地修改。当执行hexo g，就会将其中的内容生成为静态网页，并放入public文件夹下。

public 文件

   存放着生成的HTML文件内容，最终会被push到GitHub上。

themes文件

   下载的Hexo博客的主题文件，要放在该目录下，每个主题都有自己的_config.yml文件。
