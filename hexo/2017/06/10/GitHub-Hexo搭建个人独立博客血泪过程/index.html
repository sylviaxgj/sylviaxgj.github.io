GitHub-Hexo搭建个人独立博客血泪过程

发表于 2017-06-10 | 阅读次数 | 本文总阅读量 次

这是我的第一篇博客文章，想想还是有些小激动的。下面来跟大家分享一下，我搭建博客的详细过程和一些小曲折。总体来说，还算是顺利吧！

一、准备软件工具

  • git客户端

    作用：把本地的hexo内容提交到GitHub上去,我的版本是git@2.12.2（git version命令）
  • node.js
    作用：用来创建hexo博客框架的，我的版本是node.js@v6.10.3（node -v命令）
  • GitHub账号

    使用github Pages来托管GitHub上的静态网页，用它来搭建博客

二、软件安装和配置
本文是在win10系统下安装的
下面介绍详细的安装过程：

 建议先安装git。因为用node.js安装hexo的时候，可以用git。

git安装

  安装git for windows。

  安装好了之后，直接在电脑中任意一个文件夹下，右键，就可以看到如下选项：



然后，选择Git Bash Here就可以在当前路径中打开git的命令行界面了。

GitHub账号注册

  进去GitHub的官网进行注册，github传送门。

  注册过程比较简单，当然，必须记住你的邮箱和密码。注册过的童鞋请跳过，我之前就注册过了，这一步就跳过了。

创建GitHub仓库

  注册号账号以后，就是创建仓库了，用来放置你的代码文件。进入GitHub后，点击左上角的“+”号，如下：


进入到创建仓库页面，创建仓库时，只需要填写仓库名字，其他的不用管，然后点击“Create repository”即可，这里注意，仓库名一定要是yourname.github.io,比如我的GitHub昵称为sylviaxgj，则我的仓库名字就是sylviaxgj.github.io。（我之前已经创建过了，所以会有已存在警告，创建的时候是不会有的）

配置SSH Keys

检查SSH Keys

  这一步的目的是使本地git项目与远程的GitHub建立联系，使本地git提交时，拥有远程GitHub的权限，来提交到你的GitHub上。

  首先,来检查一下，你的电脑上的ssh key打：

在电脑任意文件夹处，右键git bash here，

然后在命令行输入cd ~/.ssh
来检查本机的ssh密钥

  如果提示:No such file or directory ，说明是第一次使用git。否则，会进入到.ssh文件夹，这个时候，你可以输入

pwd

来显示当前的路径

找到当前路径后，打开路径，会发现有三个文件，分别是id_rsa、id_rsa.pub、known_hosts。其中id_rsa是私钥，id_rsa.pub是公钥。可以直接删除.ssh文件夹中的所有文件，重新配置。也可以使用其中的文件。

生成新的SSH Key

  打开git，在命令行下，输入下列命令：
ssh-keygen -t rsa -C "你GitHub的注册邮箱l@example.com"，
注意：“C”为大写。

  然后，回车后，会让你输入密码，这个密码是在你每次提交项目时输入，这样可以防止别人往你的项目里提交内容。如何密码为空的话，提交项目时，不用输入密码。若是不需要密码，可以直接三次回车。

  可以输入下列命令来验证是否连接成功，ssh -T git@github.com（注意，这里就输入该命令，不用更改成自己的账户），如果显示Hi xxx! You've successfully authenticated,but GitHub does not provide shell access.，则表示连接成功，此时，已经生成新的SSH Key了，接下来，就是向GitHub中添加SSH Key了。

向GitHub中添加SSH Key

  在本地设置了SSH Key，就可以添加到GitHub上了。

  打开GitHub登录你的账户，然后点击右上角的头像，选择Settings,在显示页面的左边选择“SSH and GPG keys”，然后右上角点击“New SSH key”来创建一个SSH key。

  填写SSH key时，只用填写“Title”和“Key”两项。“Title”可以随便起名字，但是建议最好选一个反映本机相关信息的名字，表示这个SSH key是和本机相连。“Key”中填入上一步生成的id_rsa.pub中的内容，id_rsa.pub的路径一般是在C:\Users\你的电脑名.ssh文件夹下（我的是在C:\Users\xgj.ssh）。最后，点击“Add SSH key”按钮即可。
